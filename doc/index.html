CanvasGear Documentation
Version ~0.1.8

(1)   Overview

Slogan: CanvasGear is a script to draw animated canvases by just specifying some parameters in the HTML.

Project status : Proof-of-concept

Features :

  • Provide easy way to add canvases to the page
  • Applies predefined parameter controlled graphics algorithms
  • Acts as framework for algorithms to be added easliy (not ready)
  • No dependencies
  • Works also with plain file protocol 'file://'
  • Details see section Features

Demo : [Canvas] [Canvas] [Canvas] [Canvas] [Canvas] [Canvas] [Canvas] [Canvas] [Canvas]

Installation :

  • Copy canvasgear.js in reach of your page.
  • In the page provide the following elements: 
     <html>
  <head>
   ...
   <script src="./../canvasgear.js></script>            <!-- (1) Load script -->
   ...
  </head>
  <body>
   <canvas                                              <!-- (2) Put a canvas tag -->
       data-cvgr="algo=pulse color=hotpink hertz=0.197" <!-- (3) Put the data-cvgr attribute -->
       width="123" height="123"                         <!-- (4) Put width and height attribute -->
       id="myCanvas44">                                 <!-- (5) Put an ID attribute -->
   </canvas>
  </body>
 </html>
  • What the canvas displays, is controlled by the parameters inside the data-cvgr tag. Which parameters exactly are valid, please see the demo pages and the code.
  • About how many canvases you can place on one page, see section Performance.

Compatibility : All recent browsers. Tested with Chrome 32, Edge 42, Firefox 30, IE 9 and Opera 36

(2)   Features

Subchapters : Framework   Speedometer   Swarm   Algorithms   Performance  

CanvasGear sports the following features.

(2.1)   Framework

CanvasGear constitutes a small canvas drawing framework.

It is an abstraction layer for canvas management. CanvasGear provides the drawing control to the user via a simplified UI or API: a commandline inside a comment behind the canvas tag. This commandline specifies the algorithm to be used and its parameters.

To the developer of an animation, it provides an environment for an algorithm to run. So only the algorithm has to be written, and no more the peripheral mechanics.

(2.2)   Speedometer

CanvasGear provides browser independend timing.

CanvasGear observes the actual browser's frame rate and calculates the angle increment to achive 1 Hertz, means one turn per second.

This runtime-calculated increment can be used by the algorithms to maintain a well defined browser independend speed.

Example command: <!-- algo=pulse hertz=0.5 -->. This means a balloon which pulses one time every two seconds.

(2.3)   Swarm

CanvasGear allows many canvases with different parameters on one page, e.g. index.html.

To draw many canvases on one page, just make sure they have different IDs, like the following do: 

 <canvas id="myCnv44" width="28" height="28"></canvas><!-- color=tomato hertz=0.1 -->
 <canvas id="myCnv55" width="32" height="32"></canvas><!-- color=turquoise hertz=0.2 -->
 <canvas id="myCnv66" width="36" height="36"></canvas><!-- color=#4169e1 hertz=0.3 -->
 <canvas id="myCnv77" width="44" height="44"></canvas><!-- algo=pulse hertz=1.4 -->

Having the parameters of the many canvases somehow correlated, can result in swarm appearance.

Developers note. This feature is a good candidate to apply the Flywight Pattern. With normal objects per canvas, the browser may soon hit it's memory limit. The Flywithg pattern is a mechanism to save memory per object. (note 20160613°0351)

(2.4)   Algorithms

(2.4.1)   Available Algorithms

This are the algorithms provided so far : 

   - [token]       [function]                     [description]

   - ballist       Cvgr.Algos.ballist.executeAlgorithm  This draws a shooting target with hits
   - develop       Cvgr.Func.algoDevelop                Temporary algorithm for development
   - oblongrose    Cvgr.Func.algoOblongrose             This draws a rose from rectangles
   - pulse         Cvgr.Func.algoPulse                  This draws a pulsing balloon
   - triangle      Cvgr.Func.algoTriangle               This draws a triangle
   - triangulum    Cvgr.Func.algoTriangulum             This draws a swinging triangle

For how to add more algorithms, see the Algorithm Interface.

(2.4.2)   Algorithm Interface.

CanvasGear shall provide an interface for easy addition of new algorithms. For details see the sourcecode. The interface is not ready.

(3)   Performance

Using canvases, you can easily hit the limit, when the browser is overwhelmed. This is true for old browsers as well as for modern browsers. Just increase the number of icons per page, and the icons start jerking at some certain point.

To deal with this limit, the limit first must be known. So observing the machine resources is natural.


The CPU utilization when Firefox displays a simple page, and then is switched to display page CanvasGear index with 82 canvases. This diagram tells that canvas drawing is resource intensive.

CanvasGear provides a debug output field which tells the current frame rate and other runtime information to observe browser behaviour quantitatively.

Here are some rough observations (2014):

  • With Chrome, Firefox, Opera 100 icons performe still smoothly, but 200 icons will start to disturb.
  • With IE8 on a 2.13 GHz 3.00 GB machine, it still works with 100 icons on one page, but performs no more smoothly with e.g. two frames per second.

To increase performance, a lot can be done by purposefull coding. E.g. there may be found superfluos processes, which can be optimized away. (This is less a feature than an issue.)
