CanvasGear

Features

CanvasGear sports the following features.

(1)   Framework

CanvasGear constitutes a small drawing framework.

It is an abstraction layer for canvas management. CanvasGear provides the drawing control to the user via a simplified UI or API: a commandline inside a comment behind the canvas tag. This commandline specifies the algorithm to be used and its parameters.

To the developer of an animation, it provides an environment for an algorithm to run. So only the algorithm has to be written, and no more the peripheral mechanics.

(2)   Swarm

CanvasGear allows many canvases with different paramters on one page.

To draw many canvases on one page, just make sure they have different ids, like the following do: 

   <canvas id="myCanvas44" width="28" height="28"></canvas><!-- color=tomato hertz=0.1 -->
   <canvas id="myCanvas55" width="32" height="32"></canvas><!-- color=turquoise hertz=0.2 -->
   <canvas id="myCanvas66" width="36" height="36"></canvas><!-- color=#4169e1 hertz=0.3 -->

(3)   Timing

CanvasGear provides browser independend timing.

CanvasGear observes the actual browser's frame rate and calculates the angle increment to achive 1 Hertz, means one turn per second.

This runtime-calculated increment can be used by the algorithms to maintain a well defined browser independend speed.

Example: algo=pulse hertz=0.5 means, the balloon shall pulse once every two seconds.

(4)   Algorithms

This are the algorithms provided so far : 

   - develop       algoDevelop()       Temporary algorithm for development.
   - oblongrose    algoOblongrose()    This draws a rose
   - pulse         algoPulse()         This pulses.
   - triangle      algoTriangle()      This draws a triangle.
   - triangulum    algoTriangulum()    This draws a swinging triangle.

For how to add more algorithms, see the code.

(5)   Algorithm Interface.

CanvasGear provide an interface for easy addition of new algorithms. For details see the sourcecode.

(6)   Performance

Using canvases, you can easily hit the limit, when the browser is overwhelmed. This is true for older browsers as well as for most modern browsers. Just increase the number of icons per page, and they will jerk at a certain amount . To deal with this limit, observing the performance becomes necessary.

CanvasGear provides an output field which tells the current frame rate and other runtime information to observe browser behaviour quantitatively.

Here are some rough observations :

  • With Chrome, Firefox, Opera 100 icons performe still smoothly, but 200 icons will start to disturb.
  • With IE8 on a 2.13 GHz 3.00 GB machine, it still works with 100 icons on one page, but performs no more smoothly with e.g. two frames per second.

To increase performance, a lot can be done by purposefull coding. E.g. there may be found superfluos processes, which can be optimized away. (This is less a feature than an issue.)
